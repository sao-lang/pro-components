# ProTableN 技术方案

## 一、项目概述

### 1.1 目标
ProTableN 是 ProTable 的重构版本，基于 Schema 驱动的企业级表格组件。

### 1.2 核心思想
```
ProTable = DataStore + ColumnSchema + QueryForm + TableRenderer
```

### 1.3 设计原则
1. **配置化优先**：所有功能通过配置开启，降低心智负担
2. **分层架构**：核心层 + 功能层 + 插件层
3. **按需加载**：支持 Tree Shaking，未使用的功能不打包
4. **类型安全**：完整的 TypeScript 类型支持

## 二、架构设计

### 2.1 模块划分

```
ProTableN
├── Core Layer（核心层）
│   ├── DataStore        # 状态管理中心
│   ├── ColumnSchema     # 列定义管理
│   ├── RequestEngine    # 请求引擎
│   ├── QueryForm        # 查询表单
│   ├── TableRenderer    # 表格渲染
│   ├── Toolbar          # 工具栏
│   └── Pagination       # 分页
│
├── Feature Layer（功能层）
│   ├── ColumnRender     # 列渲染系统
│   ├── RowSelection     # 行选择（含跨页）
│   ├── Sorting          # 排序
│   ├── Filtering        # 筛选
│   └── Editable         # 可编辑表格
│
├── Plugin Layer（插件层）
│   ├── useUrlSync       # URL 同步
│   ├── useSearchSchema  # 查询方案
│   ├── useCache         # 缓存
│   ├── useVirtualScroll # 虚拟滚动
│   ├── useDragSort      # 拖拽排序
│   ├── useResponsive    # 响应式
│   └── useProTable      # 实例管理
│
└── Component Layer（组件层）
    ├── CardView         # 卡片视图
    ├── SkeletonTable    # 骨架屏
    ├── SearchSchemaSelector  # 查询方案选择器
    ├── ViewModeSwitch   # 视图切换
    ├── DragSortTable    # 拖拽排序表格
    └── ColumnSetting    # 列设置
```

### 2.2 数据流

```
用户操作 → DataStore → RequestEngine → 更新 DataStore → 重新渲染
```

### 2.3 Context 架构

```
RootContext（全局配置层）
  ├── props（组件属性）
  ├── rowKey（行标识）
  └── getRowKey（获取行 key 函数）

DataContext（数据状态层）
  ├── store（DataStore 实例）
  ├── formRef（表单引用）
  ├── action（操作方法）
  └── 各种状态（loading、error、dataSource 等）

ColumnContext（列配置层）
  ├── columns（列定义）
  ├── density（表格密度）
  └── onColumnsStateChange（列状态变化回调）
```

## 五、类型定义

### 5.1 核心类型

```typescript
// 列定义
interface ProColumnType<T = any> extends Omit<TableColumnProps<T>, 'render' | 'title'> {
  dataIndex?: string | string[];
  title?: ReactNode;
  valueType?: ProColumnValueType;
  valueEnum?: Record<string, { text: string; color?: string; status?: string }>;
  oprTools?: OprToolConfig<T>[];
  proTableConfig?: ProTableConfig;
  // ... 其他属性
}

// 表格属性
interface ProTableProps<T = any> extends Omit<TableProps, 'columns' | 'pagination'> {
  columns: ProColumnType<T>[];
  request?: ProTableRequest<T>;
  dataSource?: T[];

  // 查询表单
  search?: boolean | SearchConfig;

  // 工具栏
  toolbar?: ProTableToolbarConfig;

  // 行选择
  rowSelection?: ProTableRowSelectionConfig | boolean;

  // 批量操作
  batchOperation?: ProTableBatchOperationConfig;

  // 分页
  pagination?: PaginationProps | false;

  // 高级功能
  urlSync?: boolean | UrlSyncConfig;
  searchSchema?: SearchSchemaConfig;
  virtualScroll?: boolean | VirtualScrollConfig;
  dragSort?: boolean | DragSortConfig;
  editable?: EditableConfig;
  cardMode?: boolean | CardModeConfig;
  viewMode?: 'table' | 'card';
  cache?: boolean | CacheConfig;
  showSkeleton?: boolean;
  responsive?: boolean | ResponsiveConfig;
  stickyHeader?: boolean | StickyHeaderConfig;
}

// 表格操作
interface ProTableActionType {
  reload: (resetPageIndex?: boolean) => void;
  reloadAndRest: () => void;
  reset: () => void;
  clearSelected: () => void;
  setSelectedRows: (rows: any[]) => void;
  setSelectedRowKeys: (keys: (string | number)[]) => void;
  getSelectedRows: () => any[];
  getSelectedRowKeys: () => (string | number)[];
  startEditable: (rowKey: string | number) => boolean;
  cancelEditable: (rowKey: string | number) => Promise<boolean>;
  saveEditable: (rowKey: string | number) => Promise<boolean>;
  deleteEditable: (rowKey: string | number) => Promise<boolean>;
  getPagination: () => { current: number; pageSize: number; total: number };
  setPagination: (pagination: Partial<{ current: number; pageSize: number }>) => void;
  getParams: () => Record<string, unknown>;
  setParams: (params: Record<string, unknown>) => void;
  getFormInstance: () => ProFormInstance | undefined;
  startPolling: () => void;
  stopPolling: () => void;
  getPollingStatus: () => { isPolling: boolean; interval?: number };
  debouncedFetchData: (params?: Record<string, unknown>) => void;
  openDialog: <TValues = any, TRow = any>(config: OpenDialogConfig<TValues, TRow>) => DialogReturnProps;
  confirm: (config: ConfirmDialogConfig) => DialogReturnProps;
}
```

## 六、使用示例

### 6.1 基础使用

```typescript
import { ProTable } from './ProTableN';

const columns = [
  { title: '姓名', dataIndex: 'name', valueType: 'text' },
  { title: '年龄', dataIndex: 'age', valueType: 'number' },
  { title: '状态', dataIndex: 'status', valueType: 'tag', valueEnum: {
    active: { text: '启用', color: 'green' },
    inactive: { text: '禁用', color: 'red' },
  }},
  { title: '操作', valueType: 'opr', oprTools: [
    { key: 'edit', text: '编辑', onClick: (record) => console.log(record) },
    { key: 'delete', text: '删除', status: 'danger' },
  ]},
];

<ProTable
  columns={columns}
  request={async (params) => {
    const { data, total } = await fetchUsers(params);
    return { data, total };
  }}
/>
```

<ProTable
  columns={columns}

## 八、注意事项

1. **类型安全**：所有新增功能必须补充完整 TypeScript 类型
2. **性能优化**：避免不必要的重渲染，使用 memo、useMemo、useCallback
3. **错误处理**：所有异步操作必须有错误处理
4. **兼容性**：保持与旧 ProTable API 的兼容性
5. **测试覆盖**：核心功能必须编写单元测试
